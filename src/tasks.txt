CLIENT-SIDE TASKS:
  *get all pictures in an area
    - ordered by
      - likes
      - date

  *get a precise picture
    - use the previously given metadata
      - wget (or another stuff) to download the picture

  *post a picture
    - use POST HTTP
      - openID
      - get metadata of picture, insert it on the database
      - create a photo with the metadata's id in the database (like <id>.jpg)

  *like a picture

  *comment a picture



SOME HELP:

  top_lat         // TOP LEFT BORDER
  left_long       // TOP LEFT BORDER

  bottom_lat      // BOTTOM RIGHT BORDER
  right_long      // BOTTOM RIGHT BORDER


API USAGE:

  pictures_in_area?top_lat=&left_long=&bottom_lat=&right_long=
